# AI、ML、LLM 与非确定性系统专项 Profile

> 加载方式：由 [ai-review.md §6](../ai-review.md#6-风险信号--专项-profile-触发表) 触发。只补充通用维度的增量风险。

## 检视要点

- 模型输出是否作为不可信输入校验；
- 是否使用 schema、边界和确定性校验器；
- 外部文档、检索结果和工具返回是否可能注入指令；
- 系统指令、密钥和内部上下文是否可能泄露；
- 检索内容是否有来源、权限和新鲜度控制；
- 是否区分模型推断、外部信息和已确认事实；
- 缺失或低置信内容是否显式呈现；
- 高风险副作用是否有策略或人工确认；
- 工具参数、目标、权限、成本和副作用是否校验；
- 是否限制代理循环、委派深度和成本；
- 模型、提示、数据和工具版本是否可追踪；
- 是否有评测集、对抗测试和回归基线。

## 项目特定（AStockScreener）

- AIService/LiteLLM 外部 IO 方法必须挂 `@log_async_operation` 或 `@track_performance()`
- R9 日志/异常必须经 `DataSanitizer` 脱敏
- R10 禁止硬编码 API Key，必须从 `keyring` 或环境变量读取
