# 2026-07-21 Git 分支安全清理设计文档

## 1. 背景与目标
随着项目开发进展，本地和远程（GitHub 仓库）积累了大量已被合并或废弃的开发分支。本设计旨在安全地进行分支清理，遵循**“安全第一”**原则，避免误删正在使用的工作分支或未合并的开发成果。

---

## 2. 核心清理逻辑 (Core Logic)
脚本在执行时，通过以下步骤安全搜集分支状态：
1. **同步远程**：调用 `git fetch -p origin` 同步最新状态，同时自动清除远程已不存在的分支在本地的追踪引用。
2. **提取活动 Worktree 关联的分支**：运行 `git worktree list`，提取所有正在被其他工作树挂载并检出的本地分支。
3. **获取已合并分支**：
   - 本地：提取所有已合并至当前 `main` 分支的本地分支 (`git branch --merged main`)。
   - 远程：提取所有已合并至远程主分支的远程分支 (`git branch -r --merged origin/main`)。

---

## 3. 安全保护机制 (Protection Safeguards)
为防止发生误删，脚本实施多重“白名单”安全红线：
1. **主分支硬白名单**：强制保留 `main`、`master` 分支，禁止删除。
2. **活动工作树保护**：正被活动的 Git Worktree 挂载检出的分支（防止损坏工作区状态），加入白名单保护。
3. **未合并分支保护**：只针对符合 `--merged` 条件的分支列入清理范围，任何未合并的代码分支均默认保护。
4. **底层操作安全**：
   - 删除本地分支时，调用 `git branch -d <branch>`。如果分支实际上未安全合并（即便查询结果有偏差），Git 底层仍会拒绝删除并报错。
   - 仅在用户明确给出命令时，才使用 `git push origin --delete <branch>` 物理删除远程分支。

---

## 4. 交互流程设计 (UI & Confirmation)
脚本采用**全交互式命令行界面**，在进行任何写/删除操作前：
1. 控制台列出分类好的清单：
   - 🛡️ **将被保留的分支**（如 `main`、被 Worktree 占用的分支、未合并分支）以及具体原因。
   - 🗑️ **准备删除的本地分支**。
   - 🗑️ **准备删除的远程分支**。
2. 强制交互要求：提示用户输入 `yes`（必须拼写完全一致，回车）。如果输入其他任何字符（如直接回车、输入 `y` 或 `no`），则安全中止并退出，不执行删除。

---

## 5. 错误处理与容错
- 如果任意前置查询命令（如 `git status`，`git branch`，`git worktree list`）失败，脚本直接中断退出。
- 逐个执行分支删除，如遇单个分支删除失败，打印错误信息并继续，保证其他能删除的分支不被阻塞。
